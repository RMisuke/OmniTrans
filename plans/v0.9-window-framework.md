# OmniTrans v0.9 — Window Framework Architecture Design

## 1. 当前 Window Layer 架构问题

### 1.1 两套并行的窗口生命周期

| 维度 | FloatingPanel | Settings Panel |
|------|--------------|----------------|
| 类型 | `FloatingPanel: NSPanel` (自定义子类) | 普通 `NSPanel` (直接 new) |
| 创建位置 | `FloatingPanel.swift` (单例) | `App.swift:173` (内联) |
| 焦点管理 | `canBecomeKey { true }` (override) | `becomesKeyOnlyIfNeeded = false` (实例属性) |
| Dismiss | `didResignKeyNotification` → `orderOut` | `didResignKeyNotification` → `orderOut` |
| Show | `makeKeyAndOrderFront(nil)` | `makeKeyAndOrderFront(nil)` + `invalidateShadow` |
| 定位 | 鼠标附近 | 菜单栏下方居中 |
| Pin/Sticky | 支持 `isPinned` | 不支持 |
| ESC 关闭 | 支持 | 不支持 |
| 高度调整 | `updateHeight()` | 不支持 |
| 淡出动画 | `hide()` (alpha fade) | 无 |
| 拖拽 | `isMovableByWindowBackground = true` | `false` |
| `styleMask` | `[.borderless, .nonactivatingPanel]` | `[.borderless]` |
| `collectionBehavior` | `[.canJoinAllSpaces, .stationary]` | `[.canJoinAllSpaces, .transient]` |

### 1.2 具体架构问题

1. **App.swift 直接 new NSPanel**：`AppDelegate` 承载了 Window 创建、配置、定位、激活、dismiss 等全部职责，违反单一职责原则。

2. **散落的属性配置**：`styleMask`、`level`、`collectionBehavior`、`becomesKeyOnlyIfNeeded` 等关键属性在两个 panel 中分别设置，没有统一基线。

3. **不一致的 focus 机制**：`FloatingPanel` 用 `canBecomeKey` override，Settings Panel 用 `becomesKeyOnlyIfNeeded = false`。同一个问题有两种解法。

4. **Dismiss 逻辑分散**：FloatingPanel 在 `init` 中注册 `didResignKeyNotification` observer，Settings Panel 在创建时用 block-based observer。虽然机制相同，但代码重复。

5. **无窗口定位抽象**：FloatingPanel 定位到鼠标附近，Settings Panel 定位到菜单栏下方。两个定位逻辑都是内联实现的。

6. **onboardingWindow 是独立的 NSWindow**：不经过任何统一框架，直接 `NSWindow(...)` + `makeKeyAndOrderFront`。

7. **AppDelegate 存储了 3 个窗口引用**：`panel`、`settingsPanel`、`onboardingWindow`，全部由 AppDelegate 直接管理状态。

8. **v0.8.1~v0.8.4 的修补式修改**：每次都在 `toggleSettingsPanel()` 方法中内联调整属性，没有提升抽象层级。

## 2. 新 Window Framework 设计

```
┌─────────────────────────────────────────────────────────┐
│                    AppDelegate                          │
│  windowManager.toggle(.settings)                        │
│  windowManager.showFloating()                           │
└─────────────────────┬───────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────┐
│                  WindowManager                          │
│  ┌─────────────┐  ┌──────────────┐  ┌───────────────┐  │
│  │PanelFactory │  │FocusManager  │  │DismissManager │  │
│  └──────┬──────┘  └──────────────┘  └───────────────┘  │
│         │                                               │
│  ┌──────▼──────┐                                        │
│  │PositionEngine│                                       │
│  └─────────────┘                                        │
└─────────────────────────────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────┐
│                    OmniPanel                            │
│  (unified base class: canBecomeKey, styleMask, etc.)    │
│          │                     │                        │
│  ┌───────▼──────┐   ┌─────────▼─────────┐             │
│  │FloatingPanel │   │  SettingsPanel    │             │
│  │(翻译窗口)    │   │  (设置窗口)       │             │
│  └──────────────┘   └───────────────────┘             │
└─────────────────────────────────────────────────────────┘
```

### 2.1 新增文件

| 文件 | 职责 |
|------|------|
| `Sources/OmniTrans/Services/WindowManager.swift` | `WindowManager`、`PanelFactory`、`FocusManager`、`DismissManager`、`PositionEngine` |
| `Sources/OmniTrans/Services/OmniPanel.swift` | `OmniPanel` 基类（原 `FloatingPanel` 的大部分属性提升到这里） |
| `Sources/OmniTrans/Services/SettingsPanel.swift` | `SettingsPanel: OmniPanel` |

### 2.2 类职责划分

#### OmniPanel（统一基类）

```swift
class OmniPanel: NSPanel {
    // 统一配置（所有子类共享）：
    // - styleMask: [.borderless, .nonactivatingPanel]
    // - isFloatingPanel = true
    // - level = .floating
    // - collectionBehavior = [.canJoinAllSpaces, .stationary]
    // - isOpaque = false
    // - backgroundColor = .clear
    // - hasShadow = true
    // - isReleasedWhenClosed = false
    // - becomesKeyOnlyIfNeeded = false
    //
    // 统一 override：
    // - canBecomeKey { true }
    // - canBecomeMain { false }
    //
    // 统一 dismiss：
    // - didResignKeyNotification → orderOut(nil)
    //
    // 子类差异按需 override
}
```

#### FloatingPanel（翻译窗口 — 现有代码简化）

```swift
class FloatingPanel: OmniPanel {
    // 仅保留翻译窗口特有逻辑：
    // - 单例 (static let shared)
    // - isPinned → level 切换
    // - heightForMode()
    // - show(nearMouse:) — 定位到鼠标
    // - updateHeight(_:)
    // - hide() — 淡出动画
    // - 窗口拖拽状态跟踪
}
```

#### SettingsPanel（设置窗口 — 新建）

```swift
class SettingsPanel: OmniPanel {
    // 仅保留设置窗口特有逻辑：
    // - 固定尺寸 (440×500)
    // - 可更新 contentView
    // - 定位到菜单栏下方
}
```

#### WindowManager

```swift
@MainActor
final class WindowManager {
    let floatingPanel = FloatingPanel.shared
    let settingsPanel = SettingsPanel()
    
    func showFloating()         // 显示翻译窗口
    func showSettings()         // 显示设置窗口
    func hideSettings()         // 隐藏设置窗口
    func toggleSettings()       // 切换设置窗口
    func dismissAll()           // 全部关闭
}
```

#### PanelFactory

```swift
enum PanelFactory {
    static func makeFloatingPanel() -> FloatingPanel
    static func makeSettingsPanel() -> SettingsPanel
    // 未来扩展：
    // static func makeInspectorPanel() -> InspectorPanel
    // static func makePopoverPanel() -> PopoverPanel
}
```

#### FocusManager

```swift
@MainActor
struct FocusManager {
    static func makeKey(_ panel: OmniPanel)
    // 未来扩展：
    // static func restoreFocus()
    // static func chainResponder(...)
}
```

#### DismissManager

```swift
@MainActor
struct DismissManager {
    // 注册统一的 dismiss 行为到 OmniPanel
    static func attachDismissBehavior(to panel: OmniPanel)
}
```

注册 `didResignKeyNotification` observer + 可选的 ESC key handler。

#### PositionEngine

```swift
@MainActor
enum PositionEngine {
    static func positionNearMouse(_ panel: OmniPanel, width: CGFloat, height: CGFloat)
    static func positionBelowMenuBar(_ panel: OmniPanel, width: CGFloat, height: CGFloat)
    static func positionCentered(_ panel: OmniPanel)
}
```

## 4. 统一管理的 Window Property

所有以下属性在 `OmniPanel` 基类中统一设置，子类不再各自配置：

| 属性 | 统一值 | 注释 |

| `styleMask` | `[.borderless, .nonactivatingPanel]` | 统一 |
| `isFloatingPanel` | `true` | 统一 |
| `level` | `.floating` | 统一（pin 时由子类切换） |
| `collectionBehavior` | `[.canJoinAllSpaces, .stationary]` | 统一 |
| `isOpaque` | `false` | 统一 |
| `backgroundColor` | `.clear` | 统一 |
| `hasShadow` | `true` | 统一 |
| `isReleasedWhenClosed` | `false` | 统一 |
| `becomesKeyOnlyIfNeeded` | `false` | 统一 |
| `canBecomeKey` | `true` (override) | 统一 |
| `canBecomeMain` | `false` (override) | 统一 |

## 5. 共享生命周期 vs 共享实现

```
共享生命周期（OmniPanel 提供）          不共享（子类各自实现）
─────────────────────────────         ──────────────────────
- NSPanel 创建参数                    - 窗口尺寸
- Key Window 能力                     - 定位策略
- Main Window 能力                    - 内容视图
- Dismiss 行为                        - Pin/Sticky 行为
- Activation 策略                     - 淡出动画
- 窗口层级 (level)                    - 高度调整
- 焦点获得 (becomesKeyOnlyIfNeeded)   - 拖拽行为
- 窗口关闭策略
- delegate 基础行为
```

## 6. 完整的重构后调用链

```
用户点击菜单栏图标
  │
  ▼
AppDelegate.statusItemClicked()
  │
  ▼
windowManager.toggleSettings()
  │
  ├─ panel.isVisible == true?
  │    └─ YES → panel.orderOut(nil) → return
  │
  └─ panel.isVisible == false?
       │
       ├─ PanelFactory.makeSettingsPanel()  (首次)
       │    └─ OmniPanel.init()
       │         ├─ 统一属性配置
       │         ├─ DismissManager.attachDismissBehavior()
       │         └─ canBecomeKey { true }, canBecomeMain { false }
       │
       ├─ PositionEngine.positionBelowMenuBar(panel)
       │    └─ 计算菜单栏下方居中位置 → panel.setFrame(...)
       │
       ├─ FocusManager.makeKey(panel)
       │    └─ panel.makeKeyAndOrderFront(nil)
       │
       └─ (panel 显示，成为 key window，accent 正常)

用户点击窗口外部
  │
  ▼
NSWindow.didResignKeyNotification
  │
  ▼
OmniPanel.handleLostFocus()
  │
  └─ panel.orderOut(nil)
```

## 7. 实施计划

### Phase 1: 创建 OmniPanel 基类

1. 新建 `OmniPanel.swift`
2. 将 FloatingPanel 的通用属性提升到 OmniPanel
3. FloatingPanel 改为 `class FloatingPanel: OmniPanel`

### Phase 2: 创建 WindowManager + 助手类

4. 新建 `WindowManager.swift`（含 PanelFactory、FocusManager、DismissManager、PositionEngine）
5. 实现 `makeSettingsPanel()`

### Phase 3: 重构 AppDelegate

6. 在 `applicationDidFinishLaunching` 中初始化 `WindowManager`
7. 删除 `toggleSettingsPanel()`、`positionSettingsPanel()`、`showFloatingPanel()`、`showHistoryWorkspace()`
8. 替换为 `windowManager.xxx()` 调用

### Phase 4: 验证

9. 编译验证
10. 功能验证：左键菜单栏 → Settings Panel 打开 + accent 正常 + 点击外部关闭
11. 功能验证：快捷键 → Floating Panel 正常
